## Code Conventions

- **Namespaces** match folder structure: `[ProjectName].Gameplay`, `[ProjectName].UI`
- MonoBehaviours use **SerializeField** for inspector-exposed fields ‚Äî never `public` fields
- Use `[Header("...")]` to group inspector fields for readability
- Avoid `FindObjectOfType` at runtime ‚Äî inject references or use events
- `Awake` ‚Üí initialize references; `Start` ‚Üí use them; `OnEnable/OnDisable` ‚Üí subscribe/unsubscribe events

## Architecture Rules

- **No business logic in MonoBehaviours** ‚Äî keep them thin; delegate to plain C# classes
- Use **ScriptableObjects** for shared data, game events, and config (avoids singleton hell)
- Scene bootstrapping handled by a single `GameBootstrapper` or `GameManager` ‚Äî not spread across prefabs
- Use **C# events or UnityEvents** for decoupled communication ‚Äî avoid direct cross-object method calls where possible

## Performance Rules

- **Never allocate in `Update()`** ‚Äî cache references in `Awake`/`Start`
- Use object pooling for frequently spawned/destroyed objects (bullets, particles, enemies)
- Profile before optimizing ‚Äî use the Unity Profiler, not assumptions
- Avoid `string` concatenation in hot paths ‚Äî use `StringBuilder` or `string.Format`

## Scene Rules

- Each scene has one **root organizer GameObject** per category (Gameplay, UI, Environment, Managers)
- Never leave test/debug objects in a scene before committing
- Prefabs are the source of truth ‚Äî do not override prefab values in scenes without reason

## Git Rules

- Track `.meta` files ‚Äî never delete them without deleting the paired asset
- Ignore `Library/`, `Temp/`, `Logs/`, `obj/`, `Build/`
- Use Git LFS for binary assets (textures, audio, models) if repo is large

## What NOT to do

- Do not use `DontDestroyOnLoad` for everything ‚Äî use a proper scene management strategy
- Do not write game logic inside UI classes
- Do not call `GetComponent` every frame ‚Äî cache in `Awake`
- Do not use `Resources.Load` at runtime for large assets ‚Äî use Addressables

## Build

- Build targets configured in `File ‚Üí Build Settings`
- Android: set keystore via `Player Settings` (never commit keystore passwords)
- iOS: use Xcode project generated by Unity; do not hand-edit Unity-generated files
