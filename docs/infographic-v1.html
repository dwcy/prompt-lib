behind the scenes · constitution principle ii

Parallel writers need worktrees.

Two subagents writing to the same working tree silently clobber each other — git sees only the last write. The harness enforces isolation: "worktree" for every concurrent writer (Constitution v1.1.0, Gate 6).

✗ shared working tree

Silent overwrite

Two writers on the same checkout.

@react-architect @dotnet-architect 
writes App.tsx      → ok
writes App.tsx      → ok, but earlier diff lost
git status     → only second writer's content visible
✓ worktree per writer

Isolated edits

Each agent gets its own checkout + branch.

@react-architect @dotnet-architect 
isolation: "worktree"
git worktree add ../feat-fe feat/fe
git worktree add ../feat-be feat/be
→ merge cleanly at the end

behind the scenes · learned context

Two memory tiers, one rule.

Memory is reusable context, not retrained weights. Global memory is persistent across projects; project-local memory belongs to one repo. Both support hard-delete when the world changes.

Auto-memory ~/.claude/projects/.../memory/

Per-project file-based store with a MEMORY.md index. Types: user, feedback, project, reference.

  • Captures preferences, conventions, surprises
  • Never duplicates code or git history
  • Always converts relative dates to absolute
  • Stale entries deleted, not soft-marked

Self-improvement .claude/skills/self-improvement/

Project-local skill with structured memory: lessons, mistakes, preferences, evals. Triggered on corrections and post-task review.

  • ID-tagged entries: L-YYYYMMDD-NN, M-…, P-…, E-…
  • Hard removal on contradicting evidence
  • Validated by extract_lessons.py
  • Survives compaction; lives in the repo

behind the scenes · convention

Conventions Claude follows for you.

commits

Conventional + agent authorship

<type>: <subject> with type ∈ feat | task | fix | refactor | test | docs. Authored as Claude Agent <my@agent.commit> via -c flags — never modifies global git config.

branches

Refuse-on-main

The /git skill refuses to commit on main / master. Always asks for a feature branch first.

tasks.md

Phase-status convention

Every ## Phase X heading carries **Status**: ⬜🟡✅ (M/N — T###–T###). Rewritten on every checkbox change.

ADRs

Deviations have a paper trail

Cross-cutting decisions live at docs/adr/; feature-scoped ones live with the spec under specs/<feature>/contracts/.
